## 一、基于 GitHub Actions 的 CI/CD 流程（推荐）

### 1.1 流水线概览

`.github/workflows/ci.yml` 定义了三阶段自动化流水线：

```
push/PR → [test] → [docker] → [deploy（可选）]
```

| 阶段 | 任务 | 触发条件 |
|------|------|----------|
| test | Vitest 前端测试 + Maven 后端测试 + JAR 打包 | push 到 master/develop 或 PR 到 master |
| docker | 构建 Docker 镜像 + 健康检查 | test 通过 |
| deploy | SSH 部署到服务器（需手动启用） | master 分支，需配置 Secrets |

### 1.2 配置 GitHub Secrets

在 GitHub 仓库 → **Settings → Secrets and variables → Actions** 中设置：

| Secret 名称 | 用途 | 是否必填 |
|------------|------|---------|
| `DEPLOY_HOST` | 部署服务器 IP/域名 | 仅 SSH 部署 |
| `DEPLOY_USER` | SSH 登录用户名 | 仅 SSH 部署 |
| `DEPLOY_SSH_KEY` | SSH 私钥 | 仅 SSH 部署 |
| `DOCKER_USERNAME` | Docker Hub 用户名 | 如需推送镜像 |
| `DOCKER_PASSWORD` | Docker Hub 密码/Token | 如需推送镜像 |

### 1.3 启用自动部署

编辑 `.github/workflows/ci.yml`，取消 `deploy` 作业的注释：

```yaml
deploy:
  needs: [test, docker]
  runs-on: ubuntu-latest
  if: github.ref == 'refs/heads/master'
  steps:
    - uses: actions/download-artifact@v4
      with:
        name: tingchenggis-jar
    - name: SCP to server
      uses: appleboy/scp-action@v0.1.7
      with:
        host: ${{ secrets.DEPLOY_HOST }}
        username: ${{ secrets.DEPLOY_USER }}
        key: ${{ secrets.DEPLOY_SSH_KEY }}
        source: "*.jar"
        target: "/opt/tingchenggis/"
    - name: Restart service
      uses: appleboy/ssh-action@v1.0.3
      with:
        host: ${{ secrets.DEPLOY_HOST }}
        username: ${{ secrets.DEPLOY_USER }}
        key: ${{ secrets.DEPLOY_SSH_KEY }}
        script: |
          sudo systemctl restart tingchenggis
          echo "Deployment complete"
```

### 1.4 推送 Docker 镜像到仓库

如需自动推送 Docker 镜像，在 `ci.yml` 的 `docker` 作业中添加：

```yaml
- name: Log in to Docker Hub
  uses: docker/login-action@v3
  with:
    username: ${{ secrets.DOCKER_USERNAME }}
    password: ${{ secrets.DOCKER_PASSWORD }}

- name: Push to Docker Hub
  run: |
    docker tag tingchenggis:ci ${{ secrets.DOCKER_USERNAME }}/tingchenggis:latest
    docker tag tingchenggis:ci ${{ secrets.DOCKER_USERNAME }}/tingchenggis:$(date +%Y%m%d)
    docker push --all-tags ${{ secrets.DOCKER_USERNAME }}/tingchenggis
```

## 二、快速部署方式（非 CI）

### 2.1 Docker 一键启动（推荐）

```bash
git clone <repo-url> tingchenggis
cd tingchenggis
docker compose up -d
# 访问 http://localhost:8092 （H2 内存库，不需要数据库）
```

### 2.2 本地开发运行

```bash
mvn spring-boot:run
# 访问 http://localhost:8092
```

### 2.3 JAR 包手动部署

```bash
mvn clean package -DskipTests -B
java -jar target/tingchenggis-1.0.0.jar --spring.profiles.active=prod
```

## 三、配置环境变量

### 3.1 必需变量

| 变量名 | 说明 | 默认值 |
|--------|------|--------|
| `TINGCHENG_DB_PASSWORD` | PostgreSQL 密码（prod 必需） | 空 |
| `TINGCHENG_JWT_SECRET` | JWT 签名密钥，至少 32 字符 | 开发降级密钥 |
| `SERVER_PORT` | 服务端口 | 8092 |

### 3.2 可选变量

| 变量名 | 说明 |
|--------|------|
| `TINGCHENG_AI_DEEPSEEK_API_KEY` | DeepSeek API Key |
| `TINGCHENG_AI_ZHIPU_API_KEY` | 智谱 API Key |

### 3.3 设置方式

**方式 A — .env 文件（Docker 部署用）：**
```bash
cp .env.example .env
# 编辑 .env 填入实际值
```

**方式 B — Linux 环境变量：**
```bash
export TINGCHENG_DB_PASSWORD=YourStrongP@ss
export TINGCHENG_JWT_SECRET="Your-JWT-Secret-Key-At-Least-32-Chars!!"
```

**方式 C — systemd EnvironmentFile（推荐生产）：**
```ini
# /etc/tingchenggis/env
TINGCHENG_DB_PASSWORD=YourStrongP@ss
TINGCHENG_JWT_SECRET=Your-JWT-Secret-Key-At-Least-32-Chars!!
```

## 四、数据库

### 4.1 初始化 PostgreSQL

```bash
sudo -u postgres psql
```

```sql
CREATE DATABASE tingchenggis_pg;
\c tingchenggis_pg
CREATE EXTENSION postgis;
CREATE USER tingcheng WITH PASSWORD 'YourStrongP@ss';
GRANT ALL PRIVILEGES ON DATABASE tingchenggis_pg TO tingcheng;
GRANT ALL ON SCHEMA public TO tingcheng;
```

### 4.2 导入数据

启动后上传 `data/千亭.xlsx`：
```bash
curl -X POST -F "file=@data/千亭.xlsx" http://localhost:8092/thousand-pavilions/import
```

### 4.3 备份

```bash
# 每日备份 DB
pg_dump -U postgres tingchenggis_pg | gzip > /backup/db_$(date +%Y%m%d).sql.gz
# 备份上传文件
tar czf /backup/uploads_$(date +%Y%m%d).tar.gz data/uploads
# 保留 30 天
find /backup -name 'db_*.sql.gz' -mtime +30 -delete
find /backup -name 'uploads_*.tar.gz' -mtime +30 -delete
```

### 4.4 恢复

```bash
gunzip -c /backup/db_20250101.sql.gz | psql -U postgres tingchenggis_pg
tar xzf /backup/uploads_20250101.tar.gz
```

## 五、生产部署（systemd + Nginx）

### 5.1 服务文件

```ini
# /etc/systemd/system/tingchenggis.service
[Unit]
Description=TingChengGIS
After=network.target postgresql.service

[Service]
Type=simple
User=tingcheng
WorkingDirectory=/opt/tingchenggis
EnvironmentFile=/etc/tingchenggis/env
ExecStart=/usr/bin/java -jar /opt/tingchenggis/app.jar --spring.profiles.active=prod
Restart=always
RestartSec=10
SuccessExitStatus=143

[Install]
WantedBy=multi-user.target
```

```bash
sudo systemctl daemon-reload
sudo systemctl enable --now tingchenggis
```

### 5.2 Nginx 反向代理

```nginx
server {
    listen 80;
    server_name your-domain.com;
    client_max_body_size 100M;

    location / {
        proxy_pass http://127.0.0.1:8092;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        proxy_read_timeout 120s;
        proxy_send_timeout 120s;
    }
}
```

## 七、常见问题

| 问题 | 解决 |
|------|------|
| 端口 8092 被占用 | `lsof -i :8092` 查占用进程，或改 `SERVER_PORT` |
| 数据库连不上 | 检查 `pg_isready`、密码、`pg_hba.conf` |
| JDK 版本 | `java -version` 必须是 21+ |
| OSRM 超时 | 依赖公共 OSRM 服务，网络问题不影响其他功能 |
| Docker 构建慢 | `mvn dependency:go-offline` 预下载依赖，利用 Docker layer 缓存 |
| GitHub Actions 未触发 | 见下方 §7.1 |

### 7.1 GitHub Actions 未触发排查

| 可能原因 | 排查方法 | 解决 |
|----------|----------|------|
| 分支名不匹配 | `git branch` 查看默认分支名 | 修改 `.github/workflows/ci.yml` 中 `branches:` 为实际分支名 |
| 工作流文件路径不对 | 确认文件位于 `.github/workflows/ci.yml`（目录名必须是 `workflows`，复数） | 修正路径 |
| Actions 被禁用 | GitHub 仓库 → Actions 页签，检查是否显示"Workflows disabled" | 点击 "I understand my workflows, go ahead and enable them" |
| 语法错误 | Push 后立即显示黄色 ❌ | 用 `act` 本地测试，或在 GitHub 上手动编辑检查语法 |
| 无新提交 | Actions 只在 push/PR 时触发 | 推送一个新 commit 触发 |

**快速验证**：在 GitHub 仓库页面点击 **Actions** 页签 → 左侧选中 **CI** → 点击 **Run workflow** → 选择 `master` 分支 → **Run workflow** 手动触发。

## 附：文件存储路径

| 内容 | 路径 |
|------|------|
| 上传图片/视频 | `data/uploads/{日期}/{uuid}.{ext}` |
| 日志 | `logs/tingcheng.log` |
| 亭子数据 | `data/千亭.xlsx` |

> 生产环境将 `data/uploads` 挂载到持久卷（NFS / 云硬盘 / Docker volume），防止容器重启丢失。
