#### 步骤2：开发环境配置

**亭城GIS子项目开发环境要求：**

| 环境组件        | 版本要求  | 说明                            |
| ----------- | ----- | ----------------------------- |
| JDK         | 21+   | Java开发工具包，支持Spring Boot 3.1.0 |
| Maven       | 3.9+  | 项目构建工具，管理项目依赖                 |
| PostgreSQL  | 16+   | 关系型数据库，存储亭子信息                 |
| PostGIS     | 3.4+  | PostgreSQL的空间扩展，支持空间查询        |
| Spring Boot | 3.1.0 | Java应用框架，快速开发Web应用            |
| TRAE        | 最新版本  | AI辅助编程工具，提高开发效率               |

**TingChengGIS子系统环境要求：**

| 子项目类型    | 前端技术                                       | 后端技术                    | 数据库/GIS引擎                   | 特殊要求            |
| -------- | ------------------------------------------ | ----------------------- | --------------------------- | --------------- |
| WebGIS系统 | HTML5、CSS3、JavaScriptLeaflet.js/OpenLayers | Spring Boot 3.1.0       | PostgreSQL 16 + PostGIS 3.4 | 支持WMS、WFS、TMS服务 |
| 移动GIS系统  | React Native/Flutter或H5响应式设计               | Spring Boot RESTful API | PostgreSQL 16 + PostGIS 3.4 | 移动端适配、触摸交互优化    |
| 桌面GIS系统  | Electron/Tauri或JavaFX                      | Spring Boot 3.1.0       | PostgreSQL 16 + PostGIS 3.4 | 离线数据缓存、本地空间索引   |
| 三维GIS系统  | Three.js/Cesium.js或ArcGIS API              | Spring Boot + GeoServer | PostgreSQL 16 + PostGIS 3.4 | WebGL支持、3D模型加载  |
| 实时GIS系统  | WebSocketSignalR/Socket.io                 | Spring Boot 3.1.0       | PostgreSQL 16 + PostGIS 3.4 | 实时数据推送、位置跟踪     |

**环境配置步骤：**

1. 安装JDK 21，配置JAVA\_HOME环境变量
2. 安装Maven 3.9，配置MAVEN\_HOME环境变量
3. 安装PostgreSQL 16，创建数据库和用户
4. 安装PostGIS 3.4，在PostgreSQL中启用空间扩展
5. 在TRAE中配置Java和Maven路径
6. 根据选择的子系统类型，安装对应的前端框架和库
7. 测试环境：创建Spring Boot项目，连接PostgreSQL数据库

## 第二课时：空间需求分析实践（1学时）

### 2.1 需求分析方法学习

#### 步骤1：需求获取方法

1. **访谈法学习**：与亭城GIS系统用户进行深度访谈，了解空间信息需求

4. **原型法学习**：快速构建原型，验证空间交互需求

#### 步骤2：AI辅助需求提取

1. **使用Prompt提取功能列表**：基于空间分析功能，提取GIS核心功能需求

1. **完整性检查**：检查空间数据采集、处理、分析、可视化的完整性

### 附录A: TRAE教程

#### A.1 TRAE简介

TRAE是一款AI辅助编程工具，通过自然语言生成代码、优化代码、诊断错误，提高开发效率。

#### A.2 快速入门

**步骤1：安装TRAE**

1. 访问TRAE官网：https://www.trae.cn
2. 下载对应操作系统的安装包
3. 运行安装程序，按照提示完成安装
4. 启动TRAE，注册或登录账户

**步骤2：配置开发环境**

1. 在TRAE设置中配置Java开发环境
2. 设置JDK路径（例如：C:\Program Files\Java\jdk-21）
3. 配置Maven路径（例如：C:\Program Files\Maven）
4. 设置项目工作目录

**步骤3：代码生成**

1. 打开TRAE代码生成面板
2. 输入自然语言描述，例如：
   - "创建Spring Boot实体类Pavilion，包含id、name、chineseName、type等字段"
   - "生成PostGIS空间查询方法，查询指定半径范围内的亭子"
3. 选择生成的代码语言（Java）
4. 点击生成按钮，查看生成的代码
5. 复制代码到项目中

**步骤4：代码优化**

1. 选中需要优化的代码段
2. 右键选择"TRAE优化"
3. 查看优化建议，包括性能优化、代码重构建议
4. 应用优化建议

**步骤5：错误诊断**

1. 当代码出现错误时，TRAE会自动提示
2. 查看错误诊断信息
3. 根据建议修复错误
4. 重新编译验证

#### A.3 常用功能

1. **代码补全**：输入代码时自动提示补全
2. **代码生成**：通过自然语言生成完整代码
3. **代码重构**：自动重构代码结构
4. **错误修复**：自动诊断和修复代码错误
5. **代码注释**：自动生成代码注释

### 附录C: EA教程

#### C.1 EA简介

Enterprise Architect（EA）是一款专业的UML建模工具，支持用例图、类图、时序图、包图等多种UML图，适用于软件建模和系统设计。

#### C.2 快速入门

**步骤1：安装EA**

1. 访问EA官网：http://tool.uml.com.cn/ToolsEA/introduce.asp?partname=EA
2. 下载对应操作系统的安装包
3. 运行安装程序，按照提示完成安装
4. 启动EA，选择或创建项目

**步骤2：创建项目**

1. 点击"File" → "New Project"
2. 选择项目类型：Software Engineering
3. 输入项目名称：TingChengGIS
4. 选择项目存储位置
5. 创建项目视图结构

**步骤3：绘制用例图**

1. 右键项目视图 → "Add Diagram" → "Use Case Diagram"
2. 命名用例图：亭城GIS系统用例图
3. 从工具箱拖拽Actor（参与者）到图中
4. 添加参与者：普通用户、管理员、AI系统
5. 从工具箱拖拽Use Case（用例）到图中
6. 添加用例：地图浏览、亭子搜索、亭子管理、AI服务等
7. 使用Association（关联）连接参与者和用例

**步骤4：绘制类图**

1. 右键项目视图 → "Add Diagram" → "Class Diagram"
2. 命名类图：亭城GIS系统类图
3. 从工具箱拖拽Class（类）到图中
4. 添加实体类：Pavilion、User、SearchHistory
5. 为类添加属性和方法
6. 使用Association（关联）、Dependency（依赖）连接类

**步骤5：绘制时序图**

1. 右键项目视图 → "Add Diagram" → "Sequence Diagram"
2. 命名时序图：搜索亭子时序图
3. 添加对象：用户、前端、控制器、服务、数据库
4. 添加消息：输入搜索关键词 → 发送请求 → 查询数据库 → 返回结果
5. 设置消息顺序和返回值

**步骤6：绘制包图**

1. 右键项目视图 → "Add Diagram" → "Package Diagram"
2. 命名包图：亭城GIS系统包图
3. 添加包：controller、service、repository、entity
4. 使用Dependency（依赖）连接包
5. 显示包之间的层次关系

#### C.3 常用功能

1. **模型验证**：检查UML模型的正确性
2. **代码生成**：从UML模型生成代码框架
3. **文档生成**：自动生成设计文档
4. **版本控制**：集成Git进行版本管理
