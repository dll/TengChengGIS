## 1. 安全架构总览

如**图1.1**所示，安全架构分为四层：网络安全→认证授权→输入验证→数据安全，层层递进。

```
                    ┌─────────────────────────────────┐
                    │       Layer 1: 网络安全           │
                    │  HTTPS / CORS 白名单 / WAF       │
                    └─────────────┬───────────────────┘
                                  │
                    ┌─────────────┴───────────────────┐
                    │       Layer 2: 认证授权           │
                    │  JWT (HMAC-SHA256, 24h过期)      │
                    │  Spring Security RBAC           │
                    │  角色: ANONYMOUS / USER / ADMIN  │
                    └─────────────┬───────────────────┘
                                  │
                    ┌─────────────┴───────────────────┐
                    │       Layer 3: 输入验证           │
                    │  JPA参数化查询 (防SQL注入)        │
                    │  前端HTML转义 (防XSS)            │
                    │  文件类型+大小限制 (防上传攻击)    │
                    │  文件名清洗 (防路径穿越)          │
                    └─────────────┬───────────────────┘
                                  │
                    ┌─────────────┴───────────────────┐
                    │       Layer 4: 数据安全           │
                    │  BCrypt密码加密 (10轮)            │
                    │  敏感信息日志脱敏                 │
                    │  API Key存储于环境变量            │
                    └─────────────────────────────────┘
```
**图1.1 四层安全架构**：从网络层到数据层逐层防护，每层对应特定安全措施。

## 3. 身份认证与授权

### 3.1 JWT令牌认证

如**图3.1**所示，JWT认证流程为：用户登录→验证凭据→签发Token→后续请求携带Token→过滤器验证。

```java
// JwtTokenProvider.java
public String generateToken(UserDetails userDetails) {
    Date now = new Date();
    Date expiryDate = new Date(now.getTime() + 86400000); // 24小时
    return Jwts.builder()
        .subject(userDetails.getUsername())
        .issuedAt(now)
        .expiration(expiryDate)
        .signWith(getSigningKey())  // HMAC-SHA256, 1024位密钥
        .compact();
}
```
**图3.1 JWT令牌生成**：使用HMAC-SHA256签名算法，有效期为24小时。

## 5. 输入验证与防注入

如**表5.1**所示，系统针对常见注入攻击类型采取多层防护。

**表5.1 防注入防护矩阵**

| 攻击类型 | 风险等级 | 防护措施 | 实现方式 |
| --- | --- | --- | --- |
| SQL注入 | 高 | JPA参数化查询 | 使用`findByName(String)`而非拼接SQL |
| SQL注入 | 高 | 禁用Native SQL | 不写`@Query(nativeQuery=true)` |
| XSS | 中 | 前端HTML转义 | `textContent`代替`innerHTML` |
| 路径穿越 | 中 | 文件名清洗 | 移除`../`、`..\\`和特殊字符 |
| 文件上传 | 中 | 类型+大小限制 | MIME类型白名单+100MB上限 |
