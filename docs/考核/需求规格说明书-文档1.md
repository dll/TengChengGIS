**项目名称**：滁州亭城GIS系统（TingChengGIS）——多模态空间信息系统

**制定日期**：2026年6月

## 1. 项目背景与目标

### 1.1 项目背景

滁州素有"亭城"美誉，境内拥有琅琊山、醉翁亭、丰乐亭等千年名亭及各类文化亭台数百座。然而现有旅游服务系统多为通用平台，存在以下痛点：

1. **信息分散**：景点信息散布于多个平台，缺乏统一的空间信息入口
2. **文化资源数字化程度低**：大量历史文化资源未以数字化形式呈现
3. **多模态交互能力不足**：缺少基于位置的空间查询、AI问答、语音导航等智能化交互方式
4. **批量化数据管理缺失**：对千亭规模的数据缺乏高效的批量导入与管理机制

### 1.2 项目目标

构建一个以AI辅助开发为核心、融合GIS技术与多模态交互的文化旅游信息平台，实现：

1. **空间数据管理**：建立亭空间数据库，支持多源数据的存储、检索与批量管理
2. **电子地图展示**：提供2D/3D双模地图可视化，支持标记聚合、空间查询
3. **AI智能交互**：集成多模型LLM（OpenAI/DeepSeek/智谱），实现知识问答、文化内容生成
4. **TSP路线规划**：基于2-opt算法的最优游览路线求解与动画展示
5. **多模态GIS**：融合文本、语音、图像、动画、3D场景的多维空间信息表达
6. **千亭管理**：支持Excel/GeoJSON/CSV的批量导入导出

## 6. 多模态GIS特色功能

### 6.1 标记聚合（Marker Clustering）

当千亭数据加载至地图时，若标记数量超过2000，系统基于Leaflet.markercluster[11]实现空间距离聚类。聚合算法如下：

1. 将地图视图按屏幕像素划分为网格（默认大小60×60像素）
2. 同一网格内的标记合并为一个聚合点，显示数量（如"12"）
3. 缩放级别变化时自动重新计算聚类
4. 点击聚合点自动放大地图至下一级，展开细分

如**图6.1**所示，聚合效果在不同缩放级别下动态变化。

```
缩放级别Z10:        缩放级别Z14:        缩放级别Z18:
  [228]              [12] [8] [15]      醉翁亭 丰乐亭 意在亭
    ↓ 点击聚合          ↓ 细分            每个标记独立显示
```
**图6.1 标记聚合层级示意**：在低缩放级别（Z10）将所有标记聚为1簇，Z14时按空间距离分为多簇，Z18时每个亭子独立显示。

### 6.2 TSP路线规划与动画

系统内置基于2-opt局部搜索算法的TSP求解器（`TspSolver.java`），如**图6.2**所示，支持三种优化目标。

**表6.1 TSP优化目标**

| 目标 | 权重矩阵 | 适用场景 |
| --- | --- | --- |
| 最短距离 | 空间直线距离（Haversine） | 自驾游览 |
| 最少时间 | 距离÷交通速度 | 骑行/步行 |
| 最少费用 | 门票+交通成本 | 预算优先 |

路线动画流程：

1. 用户在地图选取N个亭子（N≥2）
2. 选择优化目标与交通方式
3. `TspSolver.solve()` 调用2-opt算法求解
4. 地图以逐段高亮方式绘制轨迹动画（每帧添加一段）
5. 可选GIF导出：Java 2D Graphics逐帧渲染，Animated GIF编码输出

### 6.3 语音导航（TTS）

系统在逐向导航中使用TTS（Text-to-Speech）技术实现中文语音播报，如**图6.3**所示：

```
OSRM导航API ──→ 文本路线指令 ──→ TTS引擎 ──→ 语音播报
  │                │                │
  GET /route       "前方300米       Java TTS /        扬声器
  v1/driving/      右转进入         Web Speech API
  {start};{end}    醉翁路"
```
**图6.3 语音导航流程**：OSRM返回逐向导航指令，系统将文本指令经TTS引擎合成为中文语音，经扬声器播报。支持驾车/骑行/步行三种模式。

### 6.4 3D数字地球（CesiumJS）

在2D Leaflet基础上升级至CesiumJS 3D场景，如**图6.4**所示，提供以下能力：

1. **地形展示**：全球数字高程模型叠加
2. **建筑模型**：关键景区3D白模展示
3. **飞行漫游**：沿TSP路线自动飞行浏览
4. **场景切换**：一键切换2D/3D视图

### 6.5 坐标转换（WGS-84 / GCJ-02）

系统内置坐标转换工具（`CoordinateTransform.java`），同时支持WGS-84（国际标准）和GCJ-02（中国加密）坐标体系，自动判断中国境内坐标并进行转换。
