## 1. 数据流图（DFD）

### 1.1 顶层图（上下文图）

如**图1.1**所示，系统与三类外部实体（游客、注册用户、管理员）交互，核心数据流涵盖地图浏览、查询、AI问答、路线规划、数据管理等。

```
                   ┌─────────────────────────┐
                   │     TingChengGIS系统       │
                   │                          │
   游客 ──请求──→│  地图浏览  景点查询   │──响应──→ 游客
                   │  AI问答  语音导航     │
   用户 ──请求──→│  TSP路线规划  动画展示 │──响应──→ 用户
                   │  数据采集              │
   管理员 ──请求→│  数据管理  系统配置   │──响应──→ 管理员
                   │  导入导出  权限控制   │
                   └──────────┬──────────────┘
                              │
                    ┌─────────┴─────────┐
                    │    GIS数据存储     │
                    │ (H2 / PostgreSQL  │
                    │  + PostGIS)       │
                    └───────────────────┘
```
**图1.1 系统顶层DFD**：系统为三类用户提供核心GIS与AI能力，空间数据持久化存储至数据库。

### 1.2 0层图

如**图1.2**所示，系统采用分层架构，自顶向下分为四层：控制器层、业务逻辑层、数据访问层和数据库层。

```
                        ┌─────────────┐
                        │   认证授权    │
                        │ JWT/Spring  │
                        │ Security    │
                        └──────┬──────┘
                               │
  ┌──────────┐    ┌───────────┴───────────┐    ┌──────────────┐
  │ 2D地图    │    │    业务逻辑层          │    │ 数据管理     │
  │ Leaflet  │◄──│  Service Layer        │──►│ CRUD/导入    │
  │ 标记聚合 │    │  TspSolver / AiService│    │ 导出         │
  │ CesiumJS │──►│  CoordinateTransform  │◄──│ Excel/GeoJSON│
  │ 3D地球   │    │  GeoUtils             │    │ CSV          │
  └──────────┘    └───────────┬───────────┘    └──────────────┘
                              │
                    ┌─────────┴─────────┐
                    │   数据访问层       │
                    │  JPA Repository  │
                    └─────────┬─────────┘
                              │
                    ┌─────────┴─────────┐
                    │     空间数据库     │
                    │ H2 / PG+PostGIS  │
                    │ 空间索引+WKT几何  │
                    └───────────────────┘
```
**图1.2 系统0层DFD**：展现四层架构（展示层→业务层→数据访问层→存储层）及模块间数据流向。

### 1.3 1层图——景点查询流程

如**图1.3**所示，景点查询经过控制器、服务层、Repository三层处理后返回DTO结果。

```
  用户 ──查询请求──→ PavilionController ──→ PavilionService ──→ Repository
                     │                       │                   │
                     │  GET /pavilions       │ findByType/Name    │  SQL查询
                     │  ?type=historical     │ findWithinRadius   │  参数化
                     │                       │                   │
                     └── ResponseEntity ────┘                   └──→ 数据库
                         返回DTO列表
```
**图1.3 景点查询1层DFD**：请求经Controller接收参数，Service组装查询逻辑，Repository执行参数化SQL，结果逐层返回。

### 1.4 1层图——TSP路线规划流程

如**图1.4**所示，TSP路线规划是系统的核心多模态GIS功能，调用多个服务协同完成。

```
  用户 ──选中N个亭──→ RoutePlanController ──→ ThousandPavilionsService
                     │                          │
                     │  POST /route-plans       │ 1. 获取坐标列表
                     │                          │ 2. 调用 TspSolver.solve()
                     │                          │ 3. 2-opt 算法优化
                     │                          │ 4. 生成动画帧数据
                     │                          │ 5. 可选 GIF 编码
                     │                          │
                     └── 路线结果+动画URL ←────┘
                         含 visit_order_ids       │
                          total_distance          │
                          plan_json(含动画帧)      │
```
**图1.4 TSP路线规划1层DFD**：TSP规划涉及坐标提取、2-opt求解、动画生成、GIF编码等多步操作，最终返回路线与动画数据。

**图2.1 系统ERD**：6个核心实体+1个采集记录实体，覆盖空间数据管理与业务数据管理。PavilionCollector表用于记录数据采集溯源。

## 3. 功能模块划分

如**表3.1**所示，系统分为6大功能模块，每个模块包含若干子模块。

**表3.1 功能模块划分**

| 模块 | 子模块 | 涉及GIS技术 | 核心类 |
| --- | --- | --- | --- |
| **地图模块** | Leaflet 2D显示、CesiumJS 3D显示、标记聚合、坐标转换 | WGS-84/GCJ-02转换、空间渲染 | CoordinateTransform, GeoUtils |
| **数据管理模块** | 亭子CRUD、景区管理、行政区划、批量导入导出 | 空间数据类型、WKT解析 | PavilionController, ImportService |
| **AI模块** | LLM问答（多模型轮询）、TSP路线求解、GIF动画导出 | 路径优化、空间距离计算 | AiService, TspSolver |
| **路线模块** | TSP规划、OSRM导航、GIF动画、语音导航TTS | 路网匹配、语音合成 | RoutePlanController, OSRM Client |
| **多媒体模块** | 图片展示、音频讲解、SVG渲染、3D模型 | 多媒体与空间关联 | ThousandPavilionsController |
| **用户模块** | JWT认证、RBAC权限、数据采集、旅行日志 | 数据溯源（坐标） | AuthController, AppUserService |

### 4.2 空间索引

如**图4.1**所示，空间索引用于加速大规模地理数据查询。

```sql
-- 亭子空间索引
CREATE SPATIAL INDEX idx_pavilion_geom ON pavilions(geom_wkt);

-- 景区空间索引
CREATE SPATIAL INDEX idx_scenic_area_geom ON scenic_areas(geom_wkt);

-- 行政区划空间索引
CREATE SPATIAL INDEX idx_admin_division_geom ON admin_divisions(geom_wkt);
```
**图4.1 空间索引SQL**：创建基于WKT的空间索引以优化范围查询与最近邻查询性能。

### 4.3 数据库选择策略

如**表4.3**所示，系统支持三套数据库配置，适应不同部署场景。

**表4.3 多数据库策略**

| 环境 | 数据库 | 空间支持 | 持久化 | 适用场景 |
| --- | --- | --- | --- | --- |
| 开发 | H2内存 | 有限（WKT文本） | 否 | 快速启动、单元测试 |
| 演示 | H2文件 | 有限（WKT文本） | 是（${user.home}） | 独立部署演示 |
