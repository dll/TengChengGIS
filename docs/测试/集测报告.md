## 一、测试策略

### 1.1 测试目标
- 验证系统各模块间的接口交互正确性
- 测试系统与数据库、第三方服务的集成
- 确保API接口功能正常
- 验证前后端数据交互

### 1.2 测试范围
- **API接口集成测试**：认证、亭子管理、千亭功能、空间分析、导航、VR/AR等接口
- **数据库集成测试**：JPA Repository与H2/PostgreSQL的集成
- **服务层集成**：Service与Repository、Controller的协作
- **第三方服务集成**：AI服务、路由服务等
- **前后端集成**：前端JavaScript与后端API的交互

### 1.3 集成测试策略
- 自底向上集成：先测试底层组件，再逐步向上集成
- 使用Spring Boot Test提供完整的应用上下文
- 使用H2内存数据库进行测试
- 使用MockMvc测试REST API
- 使用TestContainers（可选）进行真实数据库测试
- 使用Playwright进行端到端测试

### 1.4 测试环境
| 项 | 配置 |
|----|------|
| 数据库 | H2 (内存模式) |
| Web服务器 | 嵌入式Tomcat |
| 测试框架 | Spring Boot Test + JUnit 5 |
| E2E测试 | Playwright |

### 2.1 认证模块集成测试

#### 2.1.1 测试场景：登录流程集成

**测试组件**：
- AuthController
- AppUserService
- AppUserRepository
- JwtUtil
- SecurityConfig
- JwtAuthFilter

**测试流程**：
1. 初始化测试用户数据（通过DataInitializer或SQL脚本）
2. 客户端发送登录请求到 /auth/login
3. Controller接收请求并调用Service层
4. Service层调用Repository查询用户
5. Service层验证密码（BCrypt加密比对）
6. JwtUtil生成JWT Token
7. 返回登录响应（包含token、用户信息）

- 用户认证成功，HTTP状态码200
- JWT Token生成正确且格式有效
