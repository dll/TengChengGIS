### 4. 测试用例设计原则

本项目测试用例设计遵循以下原则：

1. **边界条件覆盖**
   - null值
   - 空字符串
   - 零值
   - 负数值
   - 极端值（最大/最小）
   - 边界值（临界值）

2. **异常场景覆盖**
   - NotFoundException
   - BusinessException
   - IllegalArgumentException
   - 权限不足场景

3. **等价类划分**
   - 有效等价类
   - 无效等价类

4. **因果图分析**
   - 条件组合测试
   - 判定表测试

### 5. 边界条件测试示例

```java
// 用户认证边界测试
@Test
void authenticate_nullUsername() {
    Optional<AppUser> result = service.authenticate(null, "password");
    assertFalse(result.isPresent());
}

// 密码长度边界测试
@Test
void register_passwordLength3() {
    assertThrows(IllegalArgumentException.class,
        () -> service.register("user", "123", "Display"));
}

// WKT解析边界测试
@Test
void parseWktBbox_singlePoint() {
    assertNull(GeoUtils.parseWktBbox("POINT(118.3 32.3)"));
}
```
