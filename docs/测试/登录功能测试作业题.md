### 题目描述
在现有的 AuthControllerTest 基础上，补充完善登录功能的单元测试用例，覆盖更多边界条件和异常场景。

### 任务要求
1. 分析 AuthController 的 login 方法，找出所有需要测试的分支逻辑
2. 补充以下测试用例（至少新增3个测试方法）：

## 作业题 2：登录功能集成测试

### 题目描述
编写登录功能的完整集成测试，验证从 Controller → Service → Repository → 数据库的完整流程。

### 任务要求
1. 使用 `@SpringBootTest` 启动完整的 Spring 应用上下文
2. 使用 H2 内存数据库进行测试
3. 测试场景包括：
   - 场景1：新用户注册 → 用户登录 → 访问受保护资源
   - 场景2：使用错误密码登录 → 收到401错误
   - 场景3：登录成功后获取当前用户信息（/auth/me）
4. 使用 TestRestTemplate 或 MockMvc 进行测试
5. 验证 JWT Token 的生成和验证功能

### 提示
- 可以使用 `DataInitializer` 或 `@Sql` 注解准备测试数据
- 需要配置测试用的 application.yml

- 测试场景完整性（40%）

## 作业题 4：登录功能自动化测试脚本开发

### 题目描述
使用 Playwright（项目已配置）开发登录功能的端到端自动化测试脚本，模拟真实用户的登录操作流程。

### 任务要求
1. 在 `e2e/` 目录下创建 `login.spec.js` 测试文件
2. 测试场景：
   - 测试1：正常用户登录流程（打开首页 → 点击登录 → 输入账号密码 → 提交 → 验证登录成功）

5. 添加合适的断言验证登录结果
