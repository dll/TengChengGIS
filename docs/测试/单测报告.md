## 一、测试计划

### 1.1 测试目标
- 验证系统各模块功能模块的单元功能正确性
- 保证代码质量符合项目规范
- 确保单元测试覆盖率达到80%以上
- 提前发现并修复潜在的Bug
- 覆盖所有边界条件和异常场景

### 1.2 测试范围
- **控制器层测试**：AuthController、PavilionController、ThousandPavilionsController、PavilionGISController、NavigationController、VrArController、AiController等
- **服务层测试**：PavilionService、AppUserService、NavigationService、ThousandPavilionsService、PavilionGISService、VrArService等
- **工具类测试**：CoordinateTransform、GeoUtils、PavilionTypeUtils、TspSolver等
- **数据访问层测试**：PavilionRepository、AppUserRepository等
- **异常处理测试**：BusinessException、NotFoundException等

### 1.3 测试策略
- 使用JUnit 5作为测试框架
- 使用Mockito进行Mock
- 使用Spring Boot Test提供集成测试支持
- 使用JaCoCo进行覆盖率统计
- 重点关注边界条件和异常场景

### 1.4 测试环境
| 项 | 配置 |
|----|------|
| 操作系统 | Windows/Linux |
| JDK版本 | 21 |
| 测试框架 | JUnit 5 + Mockito |
| 构建工具 | Maven |
