## 一、考核目标

本课程考查大作业以**AI辅助软件开发**为核心主线，要求所有子系统在需求、建模、编码、测试全流程运用 AI 辅助开发工具（TRAE / 大模型 API / Prompt Engineering），并鼓励在此基础上实现**智能体（Agent）** 与**多模态交互**等 AI 原生产品功能作为优秀档加分。具体考核以下课程目标：

**目标1（支撑毕业要求指标点 2.2）— 权重20%**

掌握通用软件工程基础理论、软件生命周期与主流过程模型，结合空间数据多源异构、时空关联、GIS集成等行业特性，熟练运用空间信息与数字技术的基本原理、模型和工程化思维，完成空间信息领域复杂工程问题的拆解、分析与规范化表达；同时结合地理信息安全、数据保密相关法规与行业标准，树立安全意识与科学严谨的工程理念。

**目标2（支撑毕业要求指标点 3.1）— 权重50%**

系统掌握空间信息系统需求分析、结构化与面向对象建模、架构设计、编码开发、功能测试、系统维护全生命周期工程化方法与技术；熟练使用EA UML、QGIS、WebGIS开发框架等专业工具开展空间数据建模、GIS架构设计与功能开发；**以AI原生思维进行系统设计，各子系统必须实现智能体（Agent）与多模态交互（文本/语音/图像/视频/AR）的深度融合**；能够运用TRAE等AI辅助开发工具完成需求梳理、建模、编码、测试等工作，提升软件开发效率；严守地理信息保密、数据安全准则与工程职业道德，具备空间信息系统独立开发与团队协作实践能力。

**目标3（支撑毕业要求指标点 11.1）— 权重30%**

理解空间信息类软件工程项目管理、成本管控、进度规划与风险预警的基本原理和方法，掌握基于EA UML的项目任务分解、进度管控等实操技能；结合GIS开发、时空数据服务、智能应急装备等行业应用场景，能够辨析工程项目在实施、运维过程中涉及的工程管理与经济决策问题，初步具备空间软件项目管理思维与综合分析能力。

## 二、大作业概述

### 2.1 项目名称

**滁州亭城GIS系统（TingChengGIS）——多模态空间信息系统开发**

### 2.2 项目背景

滁州素有"亭城"美誉，本大作业以滁州亭城文化为背景，要求学生分组完成一个完整的多模态空间信息系统的工程化开发。项目覆盖从需求分析、系统设计、编码实现、软件测试到安全评估的完整软件生命周期，全面检验学生对空间软件工程核心技能的掌握程度。

### 2.3 成绩构成

最终成绩由平时成绩（20%）、实验成绩（30%）和期末考核成绩（50%）组成，重点考核专业技能与AI应用能力的综合运用。

**平时成绩（20%）：**

- 平时作业考核（50%）：教师布置作业，根据作业内容的正确性、完整性及规范性评定成绩
- 课堂表现（20%）：教师根据学生课堂讨论、回答问题等表现评定成绩
- 阶段测验（30%）：根据测验结果评定成绩

**实验成绩（30%）：**

- 实验报告（50%）：根据实验项目完成质量及实验报告撰写情况评定成绩
- 实验作品（30%）：根据实验成果的质量与创新性评定成绩
- 小组协作（20%）：根据小组协作过程中的参与度与贡献评定成绩

**期末考核（50%）：** 采用项目设计+答辩形式，要求学生分组完成小型空间信息系统（如"滁州亭城GIS系统"）的工程化设计与编码实现，提交需求规格说明书、结构化设计说明书、面向对象设计说明书、编码实现报告、测试报告、安全分析报告、运维报告，进行现场答辩。


| 考核环节 | 权重  | 说明                                                 |
| ---- | --- | -------------------------------------------------- |
| 技术文档 | 20% | 需求规格说明书、结构化设计说明书、面向对象设计说明书、编码实现报告、测试报告、安全分析报告、运维报告 |
| 系统实现 | 15% | 系统功能完整度、代码质量、运行效果                                  |
| 答辩表现 | 10% | PPT汇报、系统演示、问答环节                                    |
| 团队协作 | 5%  | 组内分工合理、Gitee仓库提交记录、小组互评                            |


> 💡 **AI辅助开发要求（对应大纲目标2/目标3）**：本课程以 **AI 辅助软件开发** 为核心主线，所有子系统在需求分析、建模、编码、测试各环节须运用 TRAE / 大模型 API / Prompt Engineering 等 AI 工具，并在文档中留存"AI辅助开发记录"。在此基础上，鼓励有条件的小组进一步实现 **AI 原生产品功能**（智能体 Agent + 多模态交互，≥2种模态），此项作为**优秀档加分项**，深度融合者优先纳入优秀评定范围。

## 三、大作业分组与子项目

### 3.2 各子项目核心功能要求

> 💡 **AI应用要求（对齐大纲：AI辅助开发为主线，AI原生功能为加分）：**
>
> 1. **AI辅助开发（必做，目标2/目标3）** — 各组须在需求、建模、编码、测试环节运用 TRAE / 大模型 API / Prompt Engineering 等 AI 工具，并在相应文档中留存"AI辅助开发记录"（工具名称、使用场景、人工审查修改记录）
> 2. **AI原生产品功能（加分，优秀档）** — 鼓励实现至少一个智能体（如AI导游/问答/推荐Agent）并支持多模态交互（文本/语音/图像/视频/AR≥2种），实现者优先纳入优秀评定范围
> 3. **多模态LLM（建议）** — 如做多模态，建议基于多模态大语言模型（GPT-4V/Gemini/Qwen-VL等）统一实现，避免为不同模态拼凑多个独立AI服务

## 八、注意事项

### 8.1 学术诚信

1. 严禁抄袭他人代码或文档，一经发现按学校相关规定处理
2. AI工具辅助生成的代码需标注来源，并经过人工审查和修改
3. 数据采集必须真实，不得伪造空间数据
4. 每个学生需独立采集数据，填充collector_id确保数据可追溯

### 8.2 提交要求

1. 所有文档提交至泛雅平台对应作业入口
2. 所有代码提交至各组Gitee仓库
3. 迟交扣分：迟1天扣10%，迟2天扣20%，超3天不予接收
4. 文档格式统一为PDF（Markdown源文件同时提交）
5. EA UML模型需提交.eap/.eapx项目文件及导出图片
6. **个人提交**：每位组员须独立提交本人负责部分的文档和代码至泛雅平台对应作业
7. **小组合并**：各组须将个人成果合并至Gitee仓库，形成完整项目
8. **答辩准入**：全组必须完成项目全部核心功能后方可参加期末答辩，未完成者不予安排答辩

### 8.3 分组管理

1. 每组设组长1名，负责项目管理和协调
2. 组员分工在Gitee仓库README.md中明确标注
3. 组内成员互评结果影响个人最终成绩
4. 如需调整分组，须在第2周前向教师申请

### 8.4 AI工具使用规范

1. 本课程鼓励使用AI工具辅助开发，但必须注明使用方式
2. 在文档中需附"AI辅助开发记录"，包括：
  - 使用的AI工具名称及版本
  - 具体使用场景（需求分析/代码生成/测试用例生成/代码审查等）
  - AI生成内容的人工审查与修改记录
3. 禁止完全依赖AI生成的内容不经审查直接提交
